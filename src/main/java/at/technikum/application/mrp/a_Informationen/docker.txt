Um Quellcode auszuführen brauchen wir:
- Copiler
- runtime
- libraries

- Operating System mit Kernel
- Hardware

Docker fasst uns code, compiler, runtime und libraries in Containern zusammen,
damit, wenn wir den Code teilen, die Person die ihn ausführen will,
nicht alles selber zusammen sammeln muss.

Beispiel: Ablauf, wenn wir hier auf "compile" klicken:

FROM startsystem(z.B. ubuntu oder alpine)

RUN apt install Library(z.B. git)

RUN apt install Javaversion(z.B. jdk-21)

RUN apt install maven-3

COPY /src /                 //nimm diesen Source Ordner und kopiere ihn in diesen Container

RUN mvn package pom.xml

CMD ["java", "-jar", "target/...jar"]


Satt selber herauszufinden welche Javaversion wir brauchen, welchen Compiler usw,
wird von Docker einfach das Docker-Image ausgeführt.


Image zu Container wie Klasse zu Objekt

Image: Eine Vorlage oder Bauplan eines Containers.
 Enthält alles, was nötig ist, um Container zu erstellen (Code, Bibliotheken, Umgebung)

Container: Eine laufende Instanz eines Images.
 Mehrere Container können vom selben Image gestartet werden.

Wir sprechen bei Docker immer von Virtualisisierung,
es unterscheidet sich aber von virutellen Maschienen - nicht verwechseln.
(VM = Ressourcen unterteilung, Container = )
